## mcpkit.run 产品需求文档（PRD）V3

| 文档属性       | 内容                                                                        |
| ---------- | ------------------------------------------------------------------------- |
| **产品名称**   | mcpkit.run                                                                |
| **产品定位**   | AI Agent 时代的一站式 MCP 工具包发现、认证与分发平台                                         |
| **核心价值主张** | 不只帮开发者找到 MCP 工具，更提供经过验证的、可协同工作的**场景化工具包 (Kit)**，让 AI Agent 的能力像拼乐高一样简单可靠。 |
| **目标用户**   | AI Agent 开发者、独立开发者、技术型中小团队、无代码/低代码 AI 建造者                                 |
| **文档版本**   | v1.0                                                                      |
| **创建日期**   | 2026-05-13                                                                |

---

### 1. 产品背景与战略机会

- **市场爆炸但混乱**：MCP 服务器数量已超 17 万，但 67% 集中在基础开发领域，垂直行业工具稀缺且难以发现。
- **集成和信任成本极高**：开发者需要花费大量时间验证安全性、兼容性与质量，超过 95% 的企业级 Agent 项目因工具集成失败而受挫。
- **“目录”已无差异化**：现有竞品多为静态列表或基础目录，缺乏质量评价、安全扫描和场景化封装。
- **“Kit”概念填补空白**：用户需要的不是零件清单，而是可直接用于解决特定问题的**可信工具组合 + 配置方案**。

**我们的机会：** 成为 MCP 生态中第一个 **“认证工具箱平台”** ，占据“可靠、即用”的心智，从流量入口升级为分发层基础设施。

### 2. 用户画像与核心需求

| 用户类型             | 核心场景                           | 痛点                            | 核心需求                                     |
| ---------------- | ------------------------------ | ----------------------------- | ---------------------------------------- |
| **AI Agent 开发者** | 为自己的 Agent 寻找并集成工具，快速搭建功能。     | 找不到特定领域的工具；不敢随意集成陌生的 MCP 服务器。 | 需要一个按场景搜索、有安全认证的精选库；希望获得“打包好的工具组合”和配置模板。 |
| **独立开发者/小团队**    | 用 AI Agent 自动化业务流程（如电商、营销、数据）。 | 缺乏时间调研和测试；希望有付费也能接受的省心方案。     | 直接购买/订阅能解决业务问题的“Kit”，商家端有售后保障。           |
| **MCP 工具作者**     | 希望自己的工具被发现、被集成、被认可，并实现变现。      | 曝光渠道单一，无法证明可靠性；缺乏收费分发渠道。      | 需要一个有权威认证、能承载付费工具的“应用商店”，并获得分成。          |

### 3. 核心功能需求

#### 3.1 工具发现与智能目录

- **多维搜索与筛选**：支持按名称、描述、标签、分类、协议版本、GitHub stars、更新频率等客观指标筛选。
- **场景化分类导航**：打破技术分类，按“我想做什么”组织，例如“管理我的电商店铺”、“自动发布社交媒体内容”、“监控我的云服务器”。
- **工具详情页**：展示元数据、安装命令、环境变量、协议方法、Quality Score、用户评价、相似 Kit 推荐。

#### 3.2 Kit（工具包）系统——核心差异点

- **Kit 定义**：由 2-8 个 MCP 服务器组成的、面向特定场景的可协同工作配置方案。
- **Kit 组成项**：
  - 包含的 MCP 工具列表（及版本）。
  - 协同配置指南（环境变量、依赖关系）。
  - 示例 Agent 配置片段（JSON/YAML，兼容 Claude Desktop、Cursor 等）。
  - 场景描述与解决的问题。
- **Kit 类型**：
  - **官方认证 Kit**：由 mcpkit.run 团队精选、测试并打标。
  - **社区贡献 Kit**：由用户提交配方，经过社区投票和审核后上线。
- **Kit 详情页**：展示“它能做什么”的流程图、集成步骤、安全扫描结果、“一键复制配置”按钮。

#### 3.3 质量认证与信任体系

- **MCP Kit Certified 徽章**：满足以下条件的 Kit/工具可获得：
  - 代码自动化安全扫描无高危漏洞。
  - 最近 30 日内可用性测试通过（自动化回归）。
  - 文档完善度达标。
  - 作者身份验证。
- **Quality Score**（0-100 分）：根据社区活跃度、更新频率、安全评级、用户评分等综合计算的动态得分。
- **安全预警**：对已收录工具进行周期性漏洞扫描，若发现问题，立即在详情页和列表中标注风险。

#### 3.4 社区与贡献

- **提交 Kit 配方**：用户可组合工具并撰写说明，提交成为社区 Kit。
- **评价与评论**：对工具和 Kit 进行打分和评论，反馈集成体验。
- **开发者主页**：MCP 作者可认领自己的工具，展示所有作品、Kit 参与历史、获得的认证。

#### 3.5 商业化功能

- **付费推广**：CPC 广告位、按关键词的付费置顶/精选位。
- **会员订阅（Freemium）**：基础用户免费；Pro 用户解锁高级筛选、Kit 效果评测报告、无限制 API 访问；企业版提供私有化工具包托管和定制认证。
- **Kit 市场**：允许开发者上架付费 Kit，平台收取 15%-30% 佣金；未来支持订阅制工具包（类似模板市场）。

### 4. 信息架构与页面布局

#### 4.1 全局结构

1. **首页** —— 发现与入口。
2. **探索工具** —— 目录模式，支持列表和卡片视图。
3. **Kit 专区** —— 场景化工具包浏览与详情。

6. **博客/资源** —— 教程、场景实践、安全报告。

#### 4.3 探索工具 / 列表页

左侧为**场景化过滤器**：

- 场景：电商运营、DevOps、内容创作、金融、健康...
- 技术分类可选：API 调用、文件系统、数据库...
- 质量筛选：仅显示 Certified、最低 Quality Score 滑动条、是否显示免费工具。
- 排序：综合推荐、最高分、最新添加、最高人气。

右侧为工具卡片列表，每个卡片摘要：

- 名称、官方认证徽章、维护者、简短描述。
- Quality Score 圆环图、GitHub stars、集成次数。
- 悬浮即现“查看详情”和“加入 Kit 草案”按钮。

#### 4.5 开发者控制台（认领/提交）

- 通过 GitHub 登录，一键认领已收录工具。
- **提交配方**：可视化拖拽工具排序，添加连接逻辑描述，生成 Kit 配置预览。
- **认证申请**：查看自己的工具/Kit 还差哪些条件获得 Certified 徽章。
- **收入仪表盘**（未来）：查看上架 Kit 的销售和佣金，支持提现。

### 5. 内容策略

- **场景实践文章**：每创造一个官方 Kit，配发一篇《用 [Kit名称] 将 [业务] 自动化到只需要一句话》的教程，作为 SEO 长尾内容。
- **工具质量报告**：每周发布“本周高危 MCP 工具通报”，建立安全权威。
- **Kit 对决/合集**：例如“电商运营 5 大 Kit 横评”，引导用户收藏比较。
- **用户案例征集**：从社区贡献者中挖掘最佳集成故事，发放认证 Kit 权益。

---

### 6. 非功能性需求

- **SEO**：每个工具、每个 Kit 页面必须服务端渲染 (SSR)，独立的 title、description、结构化数据。
- **性能**：前置 CDN，首页 LCP < 1.5s，搜索响应 < 200ms。
- **安全**：对上传的 Kit 配方和用户提交内容进行 XSS 过滤；不存储任何用户 Agent 密钥。
- **可扩展**：数据模型需兼容未来多协议的 AI 工具（不仅仅 MCP）。

### 7. 实施路线图 (Roadmap)

**Phase 1: MVP 验证 **

- 完成网站框架、基础目录（收录 500+ 主流 MCP 工具）。
- 实现场景化分类、全文搜索和质量分展示。
- 手动构建 5 个**官方认证 Kit**（电商运营、DevOps 部署、内容营销、个人理财、健康追踪之一），配教程文章发布。
- 开放 GitHub 登录和工具提交。

**Phase 2: 社区与质量体系 **

- 上线 Kit 配方社区提交与投票功能。
- 实现自动化安全扫描和可用性回归，“Certified”徽章上线。
- 引入基础商业模块：CPC 广告、付费置顶。
- 推出 Pro 订阅（高级筛选、Kit 报告）。

**Phase 3: 市场与生态扩张**

- 正式上线 Kit 市场，支持开发者设置价格并抽取佣金。
- 开放 API，允许第三方集成 mcpkit 数据。
- 拓展到其他 AI 工具协议，成为“AI 工具包”标竿平台。
